1. Introduction

1.1 Présentation

TO-DO List est une application qui va vous permettre de lister des tâches à réaliser de manière simple et efficace. Une fois vos tâches listées, vous pourrez les gérer en les modifiant ou bien en les supprimant définitivement. TO-DO List reprends donc l’idée du fameux post-it de liste de course mais l’a adapté sous la forme d’une application web.

2. Aspects techniques

2.1 Langage

L’application a été développée en Vanilla JS, c’est à dire en pure langage JavaScript. On y retrouve donc aucune bibliothèque tierce.

2.2 Tests unitaire

Les tests unitaire ont été réalisés avec le framework Jasmine. Ils ont été réalisés selon la méthode appelée "behavior-driven development" (BDD). Le but de cette méthode étant la communication directe entre les personnes en charge du besoin et l'équipe en charge d'y répondre afin de créer un produit qui réponde parfaitement aux attentes.

2.3 Architecture

Elle a été développée à l’aide du modèle MVC (Model, View, Controller). L’idée principale du MVC est de découper l’application en trois parties interconnectées :

  • Model : son rôle est d'aller récupérer les informations « brutes » dans la base de données, de les organiser et de les assembler pour qu'elles puissent ensuite être traitées par le contrôleur
  • View : cette partie se concentre sur l'affichage. Elle se contente de récupérer des variables pour savoir ce qu'elle doit afficher.
  • Controller : c'est en quelque sorte l'intermédiaire entre le modèle et la vue : le contrôleur va demander au modèle les données, les analyser, prendre des décisions et renvoyer le texte à afficher à la vue.

2.4 Stockage

La liste de tâche sera sauvegardée en local chez l’utilisateur. Par conséquent, les tâches enregistrées sur un support spécifique ne seront disponibles que sur celui-ci.
